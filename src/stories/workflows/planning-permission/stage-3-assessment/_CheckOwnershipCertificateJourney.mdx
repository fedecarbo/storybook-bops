# Check Ownership Certificate (Assessment) — User Journey

This documents the assessment-stage review of the ownership certificate. The initial check happened during validation (Stage 1). At the assessment stage, the case officer reviews the certificate again, checks the activity log from validation, and can request a new certificate if issues are found post-validation. Two actors: **case officer** and (potentially) **applicant**.

If the certificate is correct and no further action is needed, the officer clicks **"Save and mark complete"**. The task returns to the assessment task list with a "Complete" status tag.

This is the most common outcome — the certificate was already verified during validation.

**Who:** Case officer
**Where:** Check ownership certificate → Request section

If the officer discovers an issue with the certificate during assessment (for example, new information from Land Registry reveals additional owners), they expand the **"Request new ownership certificate"** section and enter a reason explaining why the certificate type is wrong.

<Canvas of={CheckOwnershipCertificateStories.RequestExpanded} />

Unlike validation, where requests are batched and sent when the application is invalidated, assessment-stage requests are sent **immediately** to the applicant.

**Where:** Email notification → Applicant portal

The system sends an email to the applicant with the officer's reason and a link to respond. While the request is open, the request section is hidden (only one request can be active at a time). The activity log updates to show the pending request.

**Where:** External link from email → Applicant portal

The applicant receives the email, clicks the link, and either submits an updated ownership certificate or disagrees with the officer's assessment. The activity log updates to show both the validation-stage and assessment-stage request cycles.

Once the applicant responds, the request section reappears in case another request is needed.
