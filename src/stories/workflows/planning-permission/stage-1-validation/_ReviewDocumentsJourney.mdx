# Review Documents — User Journey

This documents the end-to-end journey of reviewing submitted documents during validation. The **case officer** works through each document, tagging it by category, setting visibility and consultee access, and marking it as valid or invalid.

**Who:** Case officer
**Where:** Validation task list → Review documents

The case officer opens the "Review documents" task from the validation task list. They see a table of all documents submitted by the applicant, showing the document name, any existing tags, date received, visibility, redaction status, and review status.

Each document starts as "Not started". The officer works through them one by one by clicking the document name.

Clicking a document opens the edit view. The officer sees:
- A thumbnail preview of the document on the left
- File metadata on the right (filename, date received, applicant's reason for submission)
- Below that: tag editing, document references, visibility and consultee sharing radios, and the validation question

When a document has no tags yet, the tag section shows "No tags added" with a collapsed "Edit tags" details panel:

**Who:** Case officer
**Where:** Check supplied document

The officer expands "Edit tags" to see three tabs: Drawings, Evidence, and Supporting Documents. They tick the relevant checkboxes to categorise the document.

They also:
- Add any document reference numbers (e.g. drawing numbers)
- Set whether the document should be publicly visible
- Set whether it should be shared with consultees
- Mark the document as valid or invalid
