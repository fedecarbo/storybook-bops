# Check Ownership Certificate — User Journey

This documents the end-to-end journey of checking whether the ownership certificate submitted with the application is correct. It involves **three actors**: the **case officer** (in BOPS), the **system** (automatic emails), and the **applicant** (via a secure response link).

The ownership certificate declares who owns the land: Type A (sole owner), Type B (all owners known), Type C (some known), or Type D (none known). For types B/C/D, land owners are listed with details of whether notice of the application has been served.

- **Yes, I agree** → the applicant is directed to submit a corrected ownership certificate
- **No, I don't agree** → a textarea appears for the applicant to explain why

The officer sees the response. There are three possible outcomes:

**Applicant approved** — submitted a new certificate. The officer sees the updated certificate table (in this example, the type changed from B to C with an additional land owner). If it's still incorrect, they can create another request.

<GdsCanvas of={CheckOwnershipCertificateStories.ApplicantRespondedApproved} />

**Applicant rejected** — disagreed with the officer's assessment. The officer sees the rejection reason and must decide whether to accept the original certificate or create a new request.

<GdsCanvas of={CheckOwnershipCertificateStories.ApplicantRespondedRejected} />

**No response yet** — the applicant has not responded. The request remains open until the response deadline.

The task is complete when:
- The officer confirmed the certificate was correct (selected "Yes" in step 1), or
- The applicant submitted a corrected certificate and the officer accepted it, or
- The officer reviewed the response and marked the task complete in the assessment stage
