# Add Reporting Details — User Journey

This documents the journey for adding reporting details to an application. The case officer selects the correct development type (used for national statistics returns) and answers whether the local authority owns the land. This is a **single-actor** task — only the **case officer** is involved.

**Who:** Case officer
**Where:** Validation task list → Add reporting details

The case officer opens the task from the validation task list. They see a list of reporting type radio buttons filtered to the application type. For a householder application, this typically shows Q21 — Householder developments plus any other applicable types.

Each type may include guidance text explaining what it covers, a link to further guidance on GOV.UK, and relevant legislation.

Below the reporting type selection, there is a question about local authority land ownership.

**Where:** Add reporting details form

The officer selects the development type that best matches the application. For most householder applications, this will be **Q21 — Householder developments**.

<Canvas of={AddReportingDetailsStories.WithTypeSelected} />

If the local authority is the land owner, the officer selects "Yes" and a sub-question appears asking whether the LA is carrying out the proposed works. This determines whether **Regulation 3** (LA carrying out works) or **Regulation 4** (LA not carrying out works) applies.

**Where:** Add reporting details form

If the officer tries to save without selecting a reporting type, a validation error is shown. The reporting type selection is required (when types are configured for the application type).

<Canvas of={AddReportingDetailsStories.WithErrors} />

If no reporting types are configured for the application type, the form shows an information message explaining this. The officer can still complete the task and answer the regulation questions.
