# Send Validation Decision — User Journey

This documents the end-to-end journey of validating or invalidating a planning application. It is the **final step** in the validation stage and involves **three actors**: the **case officer** (in BOPS), the **system** (automatic emails), and the **applicant/agent** (receiving notifications).

The case officer arrives here after all other validation tasks are complete. They either validate the application (moving it to consultation and assessment) or invalidate it (sending it back to the applicant with a list of issues to resolve).

If all checks passed and there are no outstanding issues, the officer clicks **"Mark the application as valid"**. The system:
1. Sets the validation date
2. Sends a validation notice email to the applicant/agent
3. Moves the application status to "In assessment"
4. Shows a success confirmation

**Who:** Case officer
**Where:** Send validation decision

If the officer has flagged issues during earlier validation tasks (e.g. requested a new document, flagged a boundary change, or raised an "other" validation request), those requests are still pending. The page shows a warning explaining that the application **cannot** be validated while requests are outstanding.

The only available action is to **invalidate** — this sends all pending requests to the applicant in one batch notification.

**Who:** System (automatic)
**To:** Applicant/agent email

When the officer clicks "Mark the application as invalid", the system sends an email to the applicant/agent. The email explains that more information is needed and provides a **secure link** to view and respond to all validation requests. The applicant has **15 business days** to respond before the application may be closed with a refund.

**Where:** Send validation decision

While the applicant is responding to the requests, the page shows the invalidated state. It confirms when the applicant was notified, displays a summary of unresolved and resolved requests with a link to view them, and explains that **all requests must be resolved or cancelled** before the application can be validated.

There is no action button in this state — the officer must wait for the applicant to respond.

**Who:** Case officer
**Where:** Send validation decision

Once the applicant has responded to all validation requests (or any remaining requests have been cancelled by the officer), the page updates. The invalidation header remains, but now the message says the application can be marked as valid. The publish radio and "Mark the application as valid" button reappear.

Note: the "valid from" date is now the **next business day after the last request was closed**, not the original received date.

The application is now validated. The applicant receives a validation email (see Step 2) and a formal validation notice is generated. The application status changes to **"In assessment"** and moves to the consultation and assessment stages.

The validation decision page becomes read-only, showing the confirmation and a link to view the formal notice.
