# Check and Request Documents — User Journey

This documents the end-to-end journey of checking submitted documents for missing items and requesting additional documents from the applicant. It involves **three actors**: the **case officer** (in BOPS), the **system** (automatic emails), and the **applicant** (responding via a secure link).

**Who:** System (automatic)
**To:** Applicant email (and agent if applicable)

When the validation requests are sent (either immediately when created, or batched as part of the validation decision), the applicant receives an email. The email tells them that the case officer needs more information and provides a secure link to view and respond to the requests.

If the applicant does not respond, the application may be closed and any payment refunded.

**Who:** Applicant
**Where:** Secure link from email (external to BOPS)

The applicant clicks the link in the email and sees a page listing all outstanding validation requests. Each additional document request appears as a task with an "Upload new document" link and a status tag.

They must respond to all requests before the case officer can proceed. Once they respond, the status changes to "Complete" and they cannot make further changes.

**Who:** Applicant
**Where:** Upload new document page

The applicant sees what document was requested and reads the case officer's reason. They have **15 business days** from the date of the request to respond — if they miss the deadline, the application may be returned with a refund.

They upload a file (PNG, JPEG, or PDF, max 30MB) and submit their response. The response is sent to the case officer immediately and cannot be changed.

**Who:** Case officer
**Where:** Check and request documents (main page)

Once the applicant has uploaded documents for all requests, the officer returns to the main page and sees:
- All requests now show a green "Closed" tag (no more "Cancel request" links)
- The newly uploaded documents appear in the submitted documents table with "Not started" status — they need to be reviewed via the **Review documents** task

The officer can now mark this task as complete and move on to reviewing the new documents.
