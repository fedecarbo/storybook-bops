# Review Validation Requests — User Journey

This documents the journey of reviewing all validation requests in one consolidated view. The "Review validation requests" page is a **read-only summary table** that shows every request type (documents, description, red line boundary, fee, ownership, other) and their current statuses. The case officer uses this page to check progress before sending the validation decision.

This page involves **one actor**: the **case officer** reviewing the status of requests created through other tasks.

The case officer opens this page from the validation sidebar to see an overview of all requests created through individual tasks (Check Description, Check Documents, Check Red Line Boundary, etc.). Before the application has been invalidated, all requests show **"Not sent yet"** (yellow tag) — they are queued and will be sent when the validation decision is made.

If no requests have been created yet, the page shows a simple empty state.

After the application is invalidated, requests are sent to the applicant automatically. The table now shows a mix of statuses as the applicant works through them:

- **"Sent"** (green) — request sent, response still within the 15-business-day deadline
- **"Overdue"** (red) — response deadline has passed
- **"Responded"** (blue) — applicant has submitted their response

This is the most common view — a mix of statuses showing the overall progress.
