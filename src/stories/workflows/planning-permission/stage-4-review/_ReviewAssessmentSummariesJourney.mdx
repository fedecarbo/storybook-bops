# Review Assessment Summaries — User Journey

This documents the **inline-review form pattern** that runs through the Review stage, focused on the "Review assessment summaries" accordion. This is the same pattern the reviewer uses on the [Review task list](?path=/docs/workflows-planning-permission-4-review-review-task-list-journey--docs); it's broken out here so designers can see the interaction in detail without the rest of the page distracting.

There are six summary categories the assessor produces — site description, summary of works, consultation summary, neighbour summary, amenity, additional evidence — and the reviewer either **agrees** with each one or **returns it with comments** for the assessor to rework.

**Where:** Review assessment summaries accordion → Site description (expanded)

When a section is expanded, the reviewer sees:

- The assessor's text wrapped in a grey-bordered **summary block** with an **Edit** link if they want to amend the assessor's work directly
- A horizontal divider
- A **footer form** in the section footer with two radios (**Agree** / **Return with comments**) and a Save submit button

The Consultation summary section additionally shows the consultee responses inline; the Amenity and Neighbour summary sections show a "View neighbour responses" link.

**Who:** Senior officer / Reviewer
**Where:** Footer form → "Return with comments" radio

Selecting **Return with comments** reveals a comment textarea below the radio. The reviewer types what needs to change and clicks **Save and mark as complete**. The section's status tag becomes **Awaiting changes** and the comment is preserved as part of the assessor's next round of work.

The reviewer has worked through every summary. Mixed verdicts are normal: some agreed, some returned with comments. Once they're done with this section they move on to the next accordion (Review conditions). The Sign-off step at the bottom of the page only unlocks once **every** section in **every** accordion is marked Agreed.
