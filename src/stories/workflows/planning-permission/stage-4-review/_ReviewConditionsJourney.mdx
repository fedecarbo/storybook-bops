# Review Conditions — User Journey

This documents the **Review conditions** accordion on the Review task list page. It contains a variable number of sub-sections depending on the application type — considerations (policies and guidance assessment), pre-commencement conditions, conditions, informatives, and rarer sub-sections for immunity / permitted-development-rights cases.

The interaction pattern is the same one used elsewhere in the Review stage (see also [Review Assessment Summaries](?path=/docs/workflows-planning-permission-4-review-review-assessment-summaries-journey--docs)): each sub-section is reviewed inline with an Agree / Return-with-comments form. The difference is the **content** the assessor produced — instead of free-form summary text, the reviewer sees structured lists of considerations, conditions, and informatives, each with a per-item Edit link.

**Who:** Senior officer / Reviewer
**Where:** Review conditions accordion → "Review assessment against policies and guidance" (expanded)

Expanded, the reviewer sees a numbered list of **considerations**. Each one shows:

- The **policy area** the assessor identified (e.g. "Design and appearance")
- The **policy** references (with optional **guidance** references)
- The assessor's **assessment**
- A **conclusion**
- An **Edit to accept** link for amending the consideration before accepting

A "Rearrange…" link below the list lets the reviewer reorder considerations. The footer form awaits a verdict on the section as a whole.

**When:** The reviewer disagrees with one or more conditions and wants the assessor to amend before sign-off.

The reviewer selects **Return with comments**, types what needs to change in the revealed comment textarea, and saves. The status tag becomes **Awaiting changes** and the comment goes back to the assessor on the next round.
