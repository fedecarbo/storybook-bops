# Site Notice v2 — Redesigned User Journey

This documents the redesigned site notice workflow. Key differences from v1:

- **Overview-first**: the landing page is the status page (or empty state), not a form
- **One page to create**: a single, clean creation form — no multi-page wizard, no massive conditional reveals
- **Single task, evolving status**: no separate "confirm" sub-task — one task that progresses through `Not started` → `Sent` → `Complete`
- **Timeline view**: the overview shows a timeline of everything that's happened and what's pending

A single, well-structured form with two sections:

1. **Notice details** — how many notices and where to display them
2. **Arrangement method** — three radios, each with a minimal conditional reveal (one field at most)

The three methods:
- **Email internal team** → shows an email input (pre-populated)
- **Email applicant** → shows a confirmation line (no extra input needed)
- **Print it myself** → shows a date input for when it will be displayed
