# Select Neighbours — User Journey

This documents how a case officer selects neighbouring addresses for consultation. Neighbours can be selected using a polygon tool on the map (which returns addresses from the OS Places API) or added manually by searching. Once addresses are confirmed, the officer proceeds to send letters.

This task involves one actor: the **case officer** in BOPS.

The case officer opens the "Select and add neighbours" task from the consultation task list. The page shows an interactive map (with the red line boundary already drawn) and collapsible instructions explaining how to select neighbours.

There are two ways to add neighbours:
1. **Draw a polygon** on the map to select all properties within the area
2. **Search manually** by typing an address or postcode

At this point, no addresses have been selected yet.

**Where:** Select and add neighbours → Map

The officer clicks and drags on the map to draw a polygon around the neighbouring properties. The system queries the OS Places API with the drawn boundary and returns all addresses found within it.

The purple area on the map shows the drawn selection polygon. A summary shows how many addresses were found (e.g. "Your search has returned 8 results"). Each address appears in a list with a "Remove" link so the officer can exclude any that aren't relevant.

The site address itself is not included in the results.

**Who:** Case officer
**Where:** Select and add neighbours → Manually add addresses

If the officer knows of specific neighbours not captured by the polygon (e.g. a property across the road, or one with a non-standard address), they can expand the "Manually add addresses" section.

An autocomplete search field lets the officer type an address or postcode. The OS Places API provides suggestions as they type. Selected addresses are added to the list alongside any map-selected addresses.

If the officer returns to this page after neighbours have already been saved, the existing addresses appear in an accordion section titled "Neighbours already selected". Each address has **Edit** and **Remove** links.

Clicking "Edit" reveals an inline text input where the officer can correct the address (e.g. fix a typo or update a flat number). They can also draw a new polygon or add more manual addresses to extend the selection.

The address entry itself shows an inline error message explaining the expected format.

From here, the officer clicks **"Continue to sending letters"** to proceed to the next task in the neighbour workflow.
