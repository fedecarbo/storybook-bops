# Consultation Task List — User Journey

This documents how a case officer works through the consultation stage of a planning application. The consultation task list is the central page for managing **three parallel workstreams**: notifying neighbours, consulting statutory bodies, and arranging publicity (site notices and press notices). The sections shown depend on the application type.

**Where:** Application overview → Consultation

After an application is validated, the case officer navigates to the consultation stage. The page shows up to three sections depending on the application type:

- **Neighbours** — selecting addresses, sending letters, reviewing responses
- **Manage consultees** — adding statutory/non-statutory consultees, sending emails, reviewing responses
- **Publicity** — arranging site notices and press notices

Below the task list is a **dates bar** showing the validation date, consultation end date, and target decision date. At the bottom, a **Details** accordion provides access to the consultation audit log.

**Where:** Consultation task list

The case officer works through the sections in parallel. Typically:

1. **Select neighbours** first — choose which addresses to notify
2. **Add consultees** — select statutory and non-statutory consultees based on constraints
3. **Send letters** and **send emails** — these become available once recipients are selected
4. Begin publicity tasks — decide if site/press notices are required

As tasks progress, their status tags update to show the current state.

**Who:** Case officer (monitoring) / Neighbours and consultees (responding externally)
**Where:** Consultation task list

Once letters and emails are sent, the response tasks show **"Awaiting response"**. The case officer monitors responses as they come in. The consultation period typically runs for 21 days.

During this waiting period, the officer can:
- View responses as they arrive
- Send follow-up emails to consultees who haven't responded
- Check the consultation end date and request an extension if needed

**Who:** Case officer / Press team
**Where:** Consultation task list → Publicity section

When a site notice or press notice is marked as **required**, a **"Confirm"** task appears below it. These confirmation tasks require evidence that the notice was actually displayed or published:

- **Confirm site notice** — enter the display date, upload a photo of the notice on site
- **Confirm press notice** — enter the publication date, upload evidence of the newspaper notice

These confirm tasks only appear once the parent notice task has been completed with "Yes, required".
