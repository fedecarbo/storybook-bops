# Press Notice v2 — Redesigned User Journey

This documents the redesigned press notice workflow. It follows the same pattern as the site notice v2 redesign:

- **Overview-first**: the landing page is the status page (or empty state), not a form
- **One page to create**: a single form with statutory reason checkboxes
- **Single task, evolving status**: no separate "confirm" sub-task — one task that progresses through `Not started` → `Requested` → `Complete`
- **Timeline view**: the overview shows a timeline of everything that's happened
