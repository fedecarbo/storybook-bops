# View Neighbour Responses — User Journey

This documents the end-to-end journey of viewing and managing neighbour responses during the consultation period. It involves **two actors**: the **case officer** (in BOPS) and **neighbours** (via the public comments portal or by post). The case officer uploads responses received offline, reviews all comments, and redacts personal data before publication.

Neighbours submit comments through the public-facing portal (linked in their notification letter). Comments received by post or email are uploaded manually by the case officer using the "Add a new neighbour response" link.

As responses come in, they appear in an accordion grouped by type:
- **Objection responses** — shown with a red status tag
- **Supportive responses** — shown with a green status tag
- **Neutral responses** — shown with a yellow status tag

Each response displays the respondent's name, email, address, date received, a truncated preview of the comment text, and any concern tags (light, privacy, noise, design, etc.).

**Who:** Case officer in BOPS
**Where:** Consultation → View neighbour responses → Add a new neighbour response

When a comment is received by post or email (not through the online portal), the case officer clicks "Add a new neighbour response" and completes the upload form:

1. **Name** and **email** of the respondent
2. **Address** — select from the list of known neighbours, or type a new address
3. **Date received** — when the comment was received
4. **Response type** — Supportive, Neutral, or An objection
5. **Tags** — categorise the concerns raised (use, privacy, light, access, noise, traffic, design, other)
6. **Response text** — the full comment (internal, not made public)
7. **Redacted response** — the version to be published (with personal data removed)
8. **Documents** — any attached letters or evidence from the respondent

**Who:** Case officer in BOPS
**Where:** Consultation → View neighbour responses → Redact comment

Before a neighbour response can be published on the planning register, the case officer must review it for personal data and redact any sensitive information. The redaction page shows:

1. **Comment submitted by** — the respondent's name, email, address, and date received
2. **Redaction guidelines** — a collapsible section explaining what needs to be redacted (personal data, names, third party addresses, contact information, special category data)
3. **Full comment** — the original text (read-only)
4. **Redacted comment** — an editable copy where the officer replaces sensitive text with `[redacted]`
5. A **"Reset comment"** link to restore the original text if needed
6. **"Save and publish"** to publish the redacted version

Once saved, the response on the main page shows "Redacted by: [officer name]" next to the "Redact and publish" link.

Once the consultation period has ended and all responses have been reviewed, redacted, and published, the task is marked as **Complete**. The page shows:

- The full count of responses grouped by type
- All responses with redaction attribution
- The "Add a new neighbour response" link is no longer shown

The consultation task list will show a green "Complete" tag against "View neighbour responses".
