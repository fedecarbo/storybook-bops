# Select Consultees (WIP) — User Journey

This documents the redesigned consultee selection flow using GOV.UK Summary Cards. Each "reason for consultation" (usually a planning constraint) is presented as a card. The case officer reviews the cards, assigns consultees, and can mark reasons as not required or add custom reasons.

This task involves one actor: the **case officer** in BOPS.

**Who:** Case officer
**Where:** Consultation task list > Add and assign consultees

The system has automatically identified planning constraints and created a **Summary Card** for each one. Cards that have a default consultee (e.g. Conservation Officer for "Conservation area") are pre-populated with the consultee's name and type (Internal/External). Cards without a suggested consultee show an "Assign consultee" link. Each card's only card-level action is "Mark as not required".

At the bottom, an "Add a reason for consultation" button allows the officer to create cards for non-constraint reasons.

If a constraint does not need consultation for this application, the officer clicks "Mark as not required" on the card. The card greys out and shows a "Not required" tag. An "Undo" link allows restoring it.

In this example, "Article 4 direction" and "Archaeological priority area" have been marked as not required.

If the officer needs to consult someone for a reason not covered by constraints (e.g. an ecology concern from the submitted documents), they click "Add a reason for consultation". An inline form appears with autocomplete search for constraint types, or they can enter a custom reason.

The officer clicks "Assign consultee" within a card. After selecting from the autocomplete search, the consultee appears inside the card as a Summary List with two rows: their name (with "Change | Remove" actions) and type (Internal/External tag). An "Add another consultee" link below allows assigning additional consultees to the same reason. A success banner confirms the assignment.
