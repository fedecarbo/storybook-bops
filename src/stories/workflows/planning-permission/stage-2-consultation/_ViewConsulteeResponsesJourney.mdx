# View Consultee Responses — User Journey

This documents the end-to-end journey of viewing and managing consultee responses during the consultation period. It involves **two actors**: the **case officer** (in BOPS) and **consultees** (statutory bodies who respond via the consultee portal or by letter/email). The case officer monitors responses as they arrive, uploads any received offline, and can redact sensitive content before publication.

After consultation emails have been sent (see "Email Consultees"), the case officer clicks "View consultee responses" from the consultation task list to monitor incoming replies. Initially, all consultees show an "Awaiting response" status tag and no filter tabs appear — only the "All" tab is visible.

Each consultee is shown in a panel card with their name, role, organisation, type badge (Internal/External), and a "Last contacted on" date.

**Who:** Consultees (via consultee portal) / Case officer (uploading offline responses)
**Where:** BOPS consultee responses page

Consultees receive an email with a link to the consultee portal where they submit their response, choosing a summary (No objection / Amendments needed / Objection) and providing text and documents. As responses come in:

- Status tags update from "Awaiting response" to the consultee's summary tag (green "No objection", yellow "Amendments needed", or red "Objection")
- Filter tabs appear for each response type that has at least one entry
- "Last received on" dates and response snippets appear on responded panels
- Consultees still awaiting show "Awaiting response" with their original "Last contacted on" date

**Who:** Case officer in BOPS
**Where:** Consultation &rarr; View consultee responses &rarr; Upload new response

When a response is received by letter, email, or phone (not through the online portal), the case officer clicks "Upload new response" on the relevant consultee's panel and completes the upload form:

1. **Name** and **email** of the person who responded
2. **Date received** — when the response was received
3. **Summary tag** — No objection, Amendments needed, or Objection
4. **Response text** — the full response (not made public)
5. **Redacted response** — a version safe for publication with personal data removed
6. **Documents** — any attached letters, reports, or evidence

**Who:** Case officer in BOPS

Clicking "View all responses" on a consultee's panel opens their detail page. This shows:

- **Summary list** with the consultee's name, role, organisation, email, consulted date, last received date, and current status
- **Response list** showing each response chronologically in an inset-text block with:
  - Received date
  - Summary tag (green "No objection" / yellow "Amendments needed" / red "Objection")
  - Published status tag (green "Published" or grey "Private")
  - Full response text (not truncated)
  - Any attached documents as download links
  - A "Redact and publish" link to prepare the response for publication
