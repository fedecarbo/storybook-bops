# Select Consultees — User Journey

This documents how a case officer adds and assigns consultees for a planning application. The system automatically suggests consultees based on the identified planning constraints (e.g. conservation area triggers a Conservation Officer suggestion). The officer reviews which constraints require consultation, checks the suggested consultees, and can manually add more via search.

This task involves one actor: the **case officer** in BOPS.

The case officer opens the "Add and assign consultees" task from the consultation task list. The page has two main sections:

1. **Constraint checkboxes** — each identified planning constraint is listed with a checkbox. The officer selects which constraints require consultation. Checkboxes auto-save when toggled.
2. **Consultee table** — shows consultees grouped by the constraint they're assigned to. The system has already pre-populated suggested consultees based on the application's constraints (e.g. a Conservation Officer is suggested for the "Conservation area" constraint).

At the bottom, a collapsed "Manually add a consultee" section allows searching for additional consultees.

**Who:** Case officer
**Where:** Add and assign consultees → Constraint checkboxes

The officer reviews the planning constraints and decides which ones actually need consultation for this particular application. Unchecking a constraint marks it as "Not required" in the consultee table — the assigned consultees remain visible but won't be contacted.

In this example, "Article 4 direction" and "Archaeological priority area" are left unchecked because they don't require consultation for this type of work.

If the officer needs to consult someone not automatically suggested (e.g. Historic England for a Grade I listed building, or a specialist ecology officer), they expand the "Manually add a consultee" section.

An autocomplete search field lets the officer type a name, organisation, or role. The system searches the contacts database and shows matching results. The officer selects a consultee and clicks "Add consultee" to add them to the table.

**Where:** Add and assign consultees → Consultee table

After clicking "Add consultee", the new consultee appears in the table under the "Other" constraint (since they were manually added, not auto-suggested for a specific constraint). Their status is "Not consulted" — they won't be emailed until the officer completes the "Email consultees" task later.

The officer can assign the consultee to a specific constraint using the "Assign consultee" links, or leave them under "Other".
