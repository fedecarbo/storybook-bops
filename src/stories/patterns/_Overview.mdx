# Patterns

Patterns used in BOPS that extend or replace GOV.UK Design System patterns for back-office planning workflows.

The [GOV.UK Design System](https://design-system.service.gov.uk/) provides excellent patterns for public-facing services where users complete tasks in a linear, step-by-step manner. BOPS is a back-office system used by planning officers who work differently — they handle complex cases non-linearly, need persistent context, and switch between tasks frequently.

These patterns document where BOPS has diverged from GDS conventions, why, and what we've learned from real users.

<hr />

## How patterns are documented

Each pattern follows the GOV.UK Design System documentation structure:

- **When to use this pattern** — the situations where this pattern is appropriate
- **When not to use this pattern** — when to use a GDS standard pattern instead
- **How it works** — structure, behaviour, and visual examples
- **Research and learnings** — what we've observed from case officers using this pattern
- **Implementation** — technical notes for developers
