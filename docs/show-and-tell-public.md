# Show and Tell — Quick Update (~2-3 min)

## Introduction

Hi everyone. For those who might not know me, I'm Fede, design lead on BOPS.

## The problem

As you know, BOPS works across various channels — there's the back office that planning officers use, but there are also emails going out to applicants, public-facing forms, notifications — and it can be quite hard to see the full picture of how all of that fits together without actually going through the whole workflow yourself.

## What we've been using

So we've been using a tool called Storybook — it's open source, and it's typically used for documenting individual components like buttons and forms and their different states.

But in our case we've put in the tasks and workflows, so you can see all the outputs — every step, every channel — in one place, without needing to set up the environment and run the application.

## Documentation

We originally set it up for the team, but it turned out to be really useful for documentation too. So we've been using it to capture how things work end to end — step-by-step journeys showing who does what and in what order, what every screen looks like at each stage, and really just leaving behind something that anyone picking this up next can browse through and understand what's been built.

## Closing

We're still finalising it, but once it's ready we'll share a link so anyone can have a browse.
